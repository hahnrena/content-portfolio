## Documentation Style Guidelines

When generating documentation, always:

- Structure information hierarchically
- Use templates and repeatable patterns
- Separate "what it is" from "how to use it"
- Include decision points and edge cases when relevant
- Reduce ambiguity in user interpretation

Avoid:
- Vague descriptions
- Overly generic explanations
- Unstructured or narrative-only documentation
